# Industry-Sprint-Back-End

## Team Setup

Each team member needs a local MySQL database. Follow these steps:

1. **Install dependencies**
   ```bash
   npm install
   ```

2. **Configure environment**
   - Copy `.env.example` to `.env`
   - Set your MySQL credentials:
     - `DB_HOST` ? MySQL host (default: localhost)
     - `DB_NAME` ? Database name (default: simplan)
     - `DB_USER` ? MySQL username
     - `DB_PASSWORD` ? MySQL password
- `JWT_SECRET` ? secret for signing tokens (use a strong value in production)

3. **Create your local database and schema**
   Run the database creation script:
   ```bash
   npm run create-db
   ```
   This creates the `simplan` database with `base_planning`, `social_economic_scores`, `financial_scores`, and `urban_impact_scores` tables, plus seed data from the CSV files in `tempdata/`.

4. **Start the server**
   ```bash
   npm start
   ```

The API runs on port 8080 by default (override with `PORT` in `.env`).

## CSV Column Definitions

The import script reads from `tempdata/` and skips definition/metadata rows embedded in the CSVs. Below are the column definitions for reference.

### Base Planning
Core zoning proposal details: building height, unit counts, floor area, transit proximity, district info, and the synthetic approval score. This is the foundational dataset all derived scores are calculated from.

### Social Economic Scores
Community-level impact indicators. All scores are relative indexes (0?100).

| Column | Definition |
|--------|-------------|
| Rent_increase_forecast_pct | Estimated percentage increase in local rents associated with the development. |
| Rent_burden_score | Relative measure of housing affordability stress. Likelihood residents will spend a high share of income on rent. |
| Displacement_risk_score | Relative estimate of potential displacement pressure. Risk that existing residents may be priced out. |
| Transit_speed_impact_score | Indicator of transit accessibility or performance impact. Higher values indicate stronger transit benefit. |

**Index ranges:** Low impact (0?30), Moderate (31?70), High (71?100).

### Financial Score
Investment and lending risk indicators derived from project characteristics.

| Column | Definition |
|--------|-------------|
| Property_value_change_pct | Estimated percentage change in surrounding property values attributable to the project. |
| Default_risk_score | Relative financial stress indicator. Higher values indicate greater project-level risk. |
| Capital_concentration_risk | Measure of how much capital exposure this single project represents. |
| Construction_loan_feasibility | Indicator of how feasible the project may be from a construction lending perspective. |
| Insurance_premium_shift_index | Relative estimate of potential change in insurance costs due to project characteristics. |

**Index ranges:** Low impact (0?30), Moderate (31?70), High (71?100).

### Urban Impact Score
Modeled physical impact projections of the proposed development.

| Column | Definition |
|--------|-------------|
| Projected_population_increase | Estimated number of new residents expected to live in the completed development. |
| Traffic_delta_pct | Estimated percentage increase in local vehicle traffic attributable to this development. |
| Environmental_stress_change | Relative increase in environmental load generated by the project. |
| Shadow_impact_score | Relative impact of building height and massing on surrounding sunlight exposure. |
| Transit_rider_impact | Relative pressure the development places on nearby public transit infrastructure. |
| Estimated_new_riders_daily | Estimated number of additional daily public transit riders generated by the project. |
